Basic

Observable

In ReactiveX an observer subscribes to an Observable. Then that observer reacts to whatever item or sequence of items the Observable emits. Use three methods to subscribe
onNext
onError
onCompleted

Observer

An observer subscribes to an Observable. An Observable emits items or sends notifications to its observers by calling the observers’ methods.

PublishSubject

PublishSubject emits to an observer only those items that are emitted by the source Observable(s) subsequent to the time of the subscription.

Single

A Single is something like an Observable, but it always either emits one value or an error notification with only use two methods to subscribe
onSuccess
onError

Scheduler

- The SubscribeOn operator designates which thread the Observable will begin operating on.
- The ObserveOn operator specifies a different Scheduler that the Observable will use to send notifications to its observers.

Varieties of Scheduler

Scheduler purpose
Schedulers.computation( ) meant for computational work such as event-loops and callback processing; do not use this scheduler for I/O (use Schedulers.io( ) instead); the number of threads, by default, is equal to the number of processors
Schedulers.from(executor) uses the specified Executor as a Scheduler
Schedulers.immediate( ) schedules work to begin immediately in the current thread
Schedulers.io( ) meant for I/O-bound work such as asynchronous performance of blocking I/O, this scheduler is backed by a thread-pool that will grow as needed
Schedulers.newThread( ) creates a new thread for each unit of work

Operators & Usages/Scenarios

Creating Observables

Operator Definition
create
create an Observable from scratch by calling observer methods programmatically
defer
do not create the Observable until the observer subscribes, and create a fresh Observable for each observer
just
convert an object or a set of objects into an Observable that emits that or those objects
from
convert some other object or data structure into an Observable

Useful Scenarios

Scenarios Code snippet
Avoid create Observable with just like below:
public Observable valueObservable() { 
    return Observable.just(value); 
}
Because just(), from(), and other Observable creation tools 
store the value of data when created, not when subscribed

Sol1:
public Observable valueObservable() { 
    return Observable.create(subscriber -> { 
        subscriber.onNext(value); 
        subscriber.onCompleted(); 
    }); 
}

Sol2:
public Observable valueObservable() { 
    return Observable.defer(
        () -> Observable.just(value)
    ); 
}
